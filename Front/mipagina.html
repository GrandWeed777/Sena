¿Que es HTML?

Html (Hypertext Markup Lenguaje) es un lenguaje de marcado estandar para la creacion de paginas web.


¿Que es un lenguaje de marcado estandar?

Html no es un lenguaje de programacion sino un sistema de etiquetas que le dice al navegador como interpretar y estructurar el contenido.


¿Que puede hacer Html?

  • Organiza el contenido: títulos, párrafos, listas, imágenes, enlaces, formularios, etc.
  • Define la estructura: encabezados, secciones, pies de página, navegación, etc.
  • No controla los estilos (eso lo hace CSS.)
  • No tiene lógica ni comportamiento (eso lo hace JavaScript)

¿Cuales son las etiquetas mas usadas?

1. Estructura basica del documento
  • < html > : Elemento raíz que contiene todo el contenido HTML de la página.
  • < head > : Aquí van los elementos que no se muestran directamente en la página, como metadatos, definición de estilos, scripts, etc.
  • < body > : Contiene el contenido visible de la página: texto, imágenes, enlaces, contenedores, etc.

2. Metadatos etiquetas dentro de (< head >)
  • < meta > : Se usa para definir metadatos como tipo de caracteres, autor, descripción y configuración de viewport.
  • < title > : Define el título que aparece en la pestaña del navegador.
  • < base > : Establece la URL base para enlaces relativos dentro del documento.
  • < link > : Relaciona recursos externos, como hojas de estilo CSS o íconos.
  • < style > : Permite insertar CSS directamente en el documento (dentro del < head >).
3. Elementos para estructurar contenido (seccionado)
  • < main > : Contiene el contenido principal de la página. Solo debe haber uno por página.
  • < header > : Contiene contenido introductorio o enlaces de navegación. Puede ir al inicio del < body >, o dentro de secciones/artículos.
  • < nav > : Bloque de enlaces de navegación (menú principal, barra lateral, etc.).
  • < section > : Agrupa contenido relacionado temáticamente, normalmente empieza con un encabezado.
  • < article > : Contenido independiente, como publicaciones de blog, noticias o comentarios.
  • < aside > : Contenido relacionado de forma tangencial, como barras laterales, anuncios o enlaces complementarios.
  • < footer > : Pie de página de un documento o sección; puede contener autoría, derechos, enlaces relacionados, etc.
4. Encabezados (niveles de jerarquía)
  • >< h1 > hasta < h6 >: Definen los títulos, desde el nivel más importante (< h1 >) hasta el menos (< h6 >), marcando la jerarquía del contenido.
5.Agrupación de texto y contenido
  • < p >Párrafo de texto.
  • < pre >Texto preformateado, respetando espacios y saltos de línea.
  • < blockquote >Cita en bloque (más extensa).
  • < hr >Línea horizontal que separa contenido.
  • < figure > Contenedor de contenido autocontenido como imágenes o diagramas.
  • < figcaption >Título o pie descriptivo para < figure >
  • < div > Contenedor genérico sin significado semántico.
  • < address >Información de contacto del autor o dueño del documento.
6.Listas
  • < ol >Lista ordenada (numerada).
  • < ul >Lista desordenada (con viñetas).
  • < li >Elemento de lista dentro de < ol > o < ul >.
  • < dl >Lista de definición.
  • < dt >Término en una lista de definición.
  • < dd >Descripción del término < dt >.
7.Texto en línea (inline) y semántica
  • < a > Enlace (hipervínculo).
  • < em > Énfasis (interpretable como entonación).
  • < strong > Importancia fuerte (semántica, no solo estilo).
  • < small > Texto de menor importancia (por ejemplo, nota al pie).
  • < s > Texto tachado (obsoleto o incorrecto).
  • < cite > Título de una obra creativa.
  • < q > Cita corta en línea.
  • < dfn > Definición de un término.
  • < abbr > Abreviatura (con atributo title explicativo).
  • < data > Asociación de texto con un valor semántico legible por máquina.
  • < time > Fecha u hora, con formato legible por máquina (atributo datetime).
  • < code > Texto con significado técnico (código).
  • < var > Variable.
  • < sam p> Salida de programa.
  • < kbd > Entrada de teclado.
  • < sub > Subíndice.
  • < sup > Superíndice.
  • < i > Estilo en cursiva sin significado semántico.
  • < b > Estilo en negrita sin significado semántico.
  • < u > Subrayado, estilístico.
  • < mark > Resaltar texto importante (marcado).
  • < bdi > Control de dirección de texto (bidireccional).
  • < bdo > Sobrescribir dirección de texto.
  • < span > Contenedor inline genérico sin semántica.
  • < br > Salto de línea.
  • < wbr > Punto opcional de quiebre de línea.
8.Tablas
  • < table > Contenedor de una tabla.
  • < caption > Título de la tabla.
  • < colgroup >, < col > Agrupación de columnas para estilos o atributos compartidos.
  • < thead >, < tbody >, < tfoot > Secciones de la tabla (cabecera, cuerpo y pie).
  • < tr > Fila de la tabla.
  • < th > Celda de cabecera.
  • < td > Celda normal.
9.Multimedia
  • < img > Imagen con atributos necesarios como src y alt.
  • < picture > Contenedor para seleccionar imágenes responsivas.
  • < source > Fuente dentro de < picture > o < video >.
  • < video >, < audio > Reproductores de contenido multimedia.
  • < track > Pistas como subtítulos o descripciones dentro de < video > o < audio >.
  • < map > Define un mapa de imagen para enlaces clicables.
  • < area > Áreas dentro del mapa definido por < map >.
10.Contenido embebido
  • < iframe > Marco embebido de otro documento HTML.
  • < embed > Incrustación de contenido externo (por ejemplo, PDFs, Flash).
  • < object > Contenedor genérico para contenido externo.
  • < param > Parámetros para contenido dentro de < object >.
11.Gráficos y scripting
  • < canvas > Lienzo para dibujo dinámico mediante JavaScript.
  • < script > Código JavaScript insertado o referenciado.
  • < noscript > Mensaje alternativo si JS está desactivado.
  • < template > Contenido para clonarse dinámicamente con JavaScript.
  • < slot > Lugar reservado en Web Components (plantillas personalizadas).
